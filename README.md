## What is Soukly?

**Soukly** (from the Arabic *سوقي* — "my market") is a multi-vendor e-commerce platform purpose-built for the **Lebanese market**. It gives small businesses, artisans, and independent brands a no-code storefront, real payment rails that actually work in Lebanon, and a curated marketplace where buyers discover them.

> One account. Buy and sell. No commissions. No middlemen taking a cut of every order.

---

## Why Soukly?

The big global marketplaces don't serve Lebanon well. Stripe, Amazon, Etsy — most either skip the country entirely or use workarounds that hurt sellers. Local solutions are fragmented across Instagram DMs, WhatsApp catalogs, and PDF price lists.

Soukly fixes that by combining four things in one place:

| Problem | The Soukly answer |
|---|---|
| Sellers stuck on Instagram with no checkout | A real storefront — products, cart, checkout, orders |
| Stripe doesn't support Lebanese accounts | **Whish Money** integration — the payment app already on every phone |
| Commission marketplaces eat 15–30% per sale | **Flat subscription** — sellers keep 100% of revenue |
| No discovery for small brands | Curated homepage, category tiles, advanced search, follow stores |

## Business & Market

### The Opportunity

Lebanon has an estimated **5.3 million people** and one of the highest mobile internet penetration rates in the region. Yet e-commerce remains **under-served**: international platforms gate Lebanese sellers out of their payment systems, and most small businesses sell via DMs and informal payment apps with no order history, no inventory, and no analytics.

The shift from Instagram-as-storefront to **dedicated commerce platforms** is happening in waves across MENA — Soukly is timed for the Lebanese wave.

### Target Market

| Segment | Size signal | Why they need Soukly |
|---|---|---|
| **Instagram boutiques** | Tens of thousands of active accounts (fashion, beauty, home) | Want a real checkout — currently lose sales to "DM us to order" friction |
| **Artisans & craft makers** | Hundreds of producers across Lebanon | Need discovery beyond their personal network |
| **Established SMBs going digital** | 50K+ registered SMEs | Need a storefront without paying for custom Shopify dev |
| **Diaspora buyers** | ~14M Lebanese diaspora | Want to buy local products & gifts for family inside Lebanon |

### Revenue Model

**Subscription, not commission.** Sellers pay a flat monthly fee. Soukly takes **zero percent of order revenue**. This is intentional:

- Predictable income for Soukly (MRR)
- Predictable cost for sellers (no surprise per-order fees)
- Higher margins for sellers vs. global marketplaces (Etsy / Amazon take 6.5%–17%)
- Aligns Soukly with seller growth, not transaction volume

### Go-to-Market

1. **Seed sellers** — direct outreach to high-quality Instagram boutiques and artisan collectives
2. **Founding seller incentives** — visible badge, homepage placement, support
3. **Buyer-side launch** — once ~50 quality stores live, push consumer marketing via Instagram, TikTok, influencer partnerships
4. **Diaspora channel** — partner with shipping aggregators so Lebanese abroad can buy and forward
5. **Vertical expansion** — once Lebanon stabilizes: Jordan, Iraq, Cyprus (similar payment-rail gaps)

### Defensibility

- **Local payment integration** (Whish) — a moat global players cannot easily cross
- **Curated catalog quality** — admin approval keeps the marketplace from becoming a spam zone
- **Network effects** — more buyers attract sellers, more sellers attract buyers
- **Subscription model** — locks in steady revenue and seller commitment

## Tech Stack

### Frontend — `frontend/`
- **Next.js 16** (App Router), **React 19**, **TypeScript**
- **Redux Toolkit + RTK Query** for state and data fetching
- **Tailwind CSS 4** + **Radix UI** components
- **Recharts** for seller analytics
- **React Hook Form + Zod** for forms

### Backend — `backend/`
- **Express 5** on Node.js
- **PostgreSQL** via **Sequelize**
- **JWT auth** (Passport + access / refresh tokens)
- **S3** via `multer-s3` for product images
- **Zod** validation, **Helmet**, **express-rate-limit** for hardening
- **Whish Money** integration for seller subscriptions

### Architecture highlights
- Paginated infinite-scroll cache pattern via RTK Query (`offset`-based merge)
- JSONB store customization (`hero`, `footer`) — sellers customize without code changes
- Two-gate seller visibility: `is_approved` (admin) + `subscription_status` (billing)
- Dual-role accounts: one user can buy and sell — no separate seller signup

## Getting Started

### Prerequisites
- Node.js 20+
- PostgreSQL 14+
- An S3-compatible bucket (optional — for product image uploads)

### Backend

```bash
cd backend
npm install
# Configure .env (DB connection, JWT secret, S3 keys)
npm run seed   # seeds demo admin / buyer / seller accounts
npm run dev    # starts on http://localhost:5000
```

### Frontend

```bash
cd frontend
npm install
# Optional: NEXT_PUBLIC_API_URL (defaults to http://localhost:5000/api/v1)
npm run dev    # starts on http://localhost:3000
```
